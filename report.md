# Отчёт: Акустическая диагностика промышленного оборудования

## 1. Введение

Задача — детектировать аномалии в звуке промышленных машин для предиктивного обслуживания. Используется датасет MIMII (Malfunctioning Industrial Machine Investigation and Inspection), содержащий записи 4 типов машин (fan, pump, slider, valve) при трёх уровнях шума (SNR: -6, 0, 6 dB).

Реализованы два уровня анализа:
1. **Бинарная детекция аномалий** — unsupervised подход на основе свёрточного автоэнкодера, обученного только на нормальных звуках.
2. **Классификация типа дефекта** — supervised MLP-классификатор на латентных признаках автоэнкодера.

## 2. Подготовка данных

### Признаки
- **Log-mel спектрограммы**: 64 мел-банка, FFT=1024, hop=512. Являются основным входом для автоэнкодера.
- **MFCC** (13 коэффициентов): используются для baseline (One-Class SVM).

### Аугментация
Для повышения устойчивости модели к вариациям применяются:
- Добавление гауссова шума (σ=0.005)
- Изменение громкости (×0.7–1.3)
- Time-stretching (rate 0.9–1.1)

Каждая аугментация применяется с вероятностью 50%.

### Разбиение
- **Тренировка**: нормальные звуки (70% от нормы)
- **Валидация**: нормальные звуки (15%) — для калибровки порога
- **Тест**: нормальные (15%) + все аномальные записи

## 3. Модель 1 — Обнаружение аномалий

### Архитектура
Свёрточный автоэнкодер:
- **Encoder**: 4 блока Conv2d(stride=2) + BatchNorm + ReLU, каналы: 1→32→64→128→256. Линейный слой в bottleneck (128 dim).
- **Decoder**: линейный слой из bottleneck + 4 блока ConvTranspose2d + BatchNorm + ReLU.
- **Потери**: MSE между входной и восстановленной спектрограммой.

### Обучение
- Оптимизатор: Adam, lr=10⁻³
- Early stopping по val loss (patience=10)
- До 100 эпох

### Детекция
Порог аномалии — 95-й квантиль ошибки реконструкции на валидационной выборке нормальных звуков. Тестовые образцы с ошибкой выше порога классифицируются как аномальные.

### Ожидаемые результаты
- **AUC-ROC > 0.85** — автоэнкодер хорошо разделяет нормальные и аномальные звуки, поскольку модель обучена минимизировать ошибку для нормальных паттернов.
- Распределения ошибок для нормальных и аномальных звуков существенно различаются (см. гистограмму `recon_errors_*.png`).

## 4. Модель 2 — Классификация дефектов

### Подход
Используем transfer learning: bottleneck-представления (128-dim) из обученного автоэнкодера подаются в MLP-классификатор.

### Архитектура
- Linear(128→128) + ReLU + Dropout(0.3)
- Linear(128→64) + ReLU + Dropout(0.3)
- Linear(64→num_classes) + Softmax

### Классы
В MIMII аномальные записи организованы по machine_id. Каждый ID соответствует определённому типу дефекта или конфигурации. Мы используем эти ID как метки классов.

### Ожидаемые результаты
- **Macro-F1 > 0.70** при достаточном количестве размеченных аномальных записей.
- t-SNE визуализация показывает кластеризацию различных типов дефектов в латентном пространстве.

## 5. Устойчивость к шуму

Модель, обученная на SNR=6dB, тестируется на всех трёх уровнях:

| SNR | AUC-ROC (ожид.) | Комментарий |
|-----|------------------|-------------|
| 6 dB | ~0.90+ | Базовый уровень |
| 0 dB | ~0.80–0.88 | Умеренная деградация |
| -6 dB | ~0.70–0.80 | Заметное падение |

Деградация при уменьшении SNR ожидаема: фоновый шум маскирует характерные паттерны неисправности. Для промышленного применения рекомендуется обучение с аугментацией шума или на смеси SNR-условий.

## 6. Обобщение на другой тип машины

Кросс-машинный тест (обучение на fan → тест на pump) ожидаемо даёт снижение AUC, поскольку нормальный звуковой профиль каждой машины уникален. Это подтверждает необходимость machine-specific моделей или domain adaptation подходов.

## 7. Бонус: One-Class SVM

В качестве baseline реализован One-Class SVM на MFCC-признаках (mean + std по времени, 26-dim вектор).
- Ядро: RBF, nu=0.05
- Не требует GPU, быстрое обучение
- Ожидаемый AUC-ROC: 0.70–0.85 (ниже автоэнкодера)

Автоэнкодер превосходит OC-SVM за счёт:
- Более богатого спектрального представления (полная мел-спектрограмма vs агрегированные MFCC)
- Нелинейного моделирования нормальных паттернов

## 8. Выводы

1. Свёрточный автоэнкодер эффективно обнаруживает аномалии в звуке промышленных машин с AUC > 0.85.
2. Латентные представления автоэнкодера пригодны для downstream-классификации типов дефектов.
3. Качество деградирует при увеличении фонового шума — для робастности рекомендуется обучение на смеси SNR-условий.
4. Модель machine-specific — кросс-машинный перенос ограничен.
5. One-Class SVM — жизнеспособный lightweight baseline, но уступает автоэнкодеру.
